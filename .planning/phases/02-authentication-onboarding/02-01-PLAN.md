**1. Create `SecureLocalStorage` (`lib/features/auth/data/secure_local_storage.dart`):**
Implement a class `SecureLocalStorage extends LocalStorage` (from `supabase_flutter`) wrapping `FlutterSecureStorage` with `AndroidOptions(encryptedSharedPreferences: true)`. Implement all required methods: `initialize()`, `hasAccessToken()`, `accessToken()`, `removePersistedSession()`, `persistSession(String)`. Use a single key `supabase_session` for storage. See RESEARCH Pattern 1 for exact implementation.

**2. Wire SecureLocalStorage into `main.dart`:**
In the `Supabase.initialize()` call (already created in Phase 1), add `authOptions: FlutterAuthClientOptions(localStorage: SecureLocalStorage())`. Do NOT remove existing Supabase initialization code — only add the `authOptions` parameter. If `Supabase.initialize()` does not exist yet, create the full initialization with `url` and `anonKey` from environment/config, plus the `authOptions`.

**3. Create auth state sealed class (`lib/features/auth/domain/auth_state.dart`):**
Create a sealed class `AppAuthState` with three variants: `authenticated(User user)`, `unauthenticated()`, `loading()`. Use Dart 3 sealed class syntax. Import `User` from `supabase_flutter`.

**4. Create `AuthRepository` (`lib/features/auth/data/auth_repository.dart`):**
Thin wrapper around `Supabase.instance.client.auth`. Methods:
- `Future<AuthResponse> signUp({required String email, required String password})` — calls `supabase.auth.signUp()`
- `Future<AuthResponse> signIn({required String email, required String password})` — calls `supabase.auth.signInWithPassword()`
- `Future<void> signOut()` — calls `supabase.auth.signOut()`
- `Future<void> resetPasswordForEmail(String email)` — calls `supabase.auth.resetPasswordForEmail(email, redirectTo: 'io.mealmate.app://reset-password')`
- `Future<UserResponse> updatePassword(String newPassword)` — calls `supabase.auth.updateUser(UserAttributes(password: newPassword))`
- `Stream<AuthState> get onAuthStateChange` — returns `supabase.auth.onAuthStateChange`
- `Session? get currentSession` — returns `supabase.auth.currentSession`
- `User? get currentUser` — returns `supabase.auth.currentUser`

All `supabase.auth.*` calls MUST go through this repository. Widgets and notifiers never call Supabase directly.

**5. Create auth state notifier (`lib/features/auth/presentation/auth_notifier.dart`):**
Using `@riverpod` annotation (Riverpod 3.x code generation pattern, NOT legacy `StateNotifier`):
- `authRepositoryProvider` — provides `AuthRepository` instance
- `authStateChangesProvider` — `@riverpod Stream<AuthState> authStateChanges(...)` wrapping `authRepository.onAuthStateChange`
- `currentUserProvider` — `@riverpod User? currentUser(...)` deriving from `authStateChangesProvider`, returning `authState.valueOrNull?.session?.user`

Run `dart run build_runner build --delete-conflicting-outputs` after creating the file to generate the `.g.dart` file.

**ANTI-PATTERNS TO AVOID:**
- Do NOT use `SharedPreferences` for session tokens — always `SecureLocalStorage`
- Do NOT use legacy `StateNotifier` — use `@riverpod` annotation with `Notifier`/`AsyncNotifier`
- Do NOT navigate from `onAuthStateChange` handler — navigation is handled exclusively by go_router redirect (Plan 02-02)

**1. Create `LoginScreen` (`lib/features/auth/presentation/screens/login_screen.dart`):**
- `Form` with `GlobalKey<FormState>` wrapping email and password `TextFormField`s
- Email field: `keyboardType: TextInputType.emailAddress`, validator checks non-empty and valid email format (`RegExp(r'^[^@]+@[^@]+\.[^@]+')`)
- Password field: `obscureText: true`, validator checks minimum 6 characters
- "Log In" `ElevatedButton` — on press: validate form, then call `ref.read(authRepositoryProvider).signIn(email: ..., password: ...)`
- Wrap the signIn call in try/catch. On `AuthException`, show error message via `ScaffoldMessenger.of(context).showSnackBar()`. Common errors: "Invalid login credentials", "Email not confirmed"
- "Don't have an account? Sign up" `TextButton` — navigates to `/signup`
- "Forgot password?" `TextButton` — navigates to `/forgot-password`
- Show a loading indicator (`CircularProgressIndicator`) while the auth call is in progress (use a local `ValueNotifier<bool>` or `useState` for loading state)

**2. Create `SignupScreen` (`lib/features/auth/presentation/screens/signup_screen.dart`):**
- Same form structure as LoginScreen but with "Sign Up" button
- Add a confirm password field with validator checking it matches the password field
- On successful signup, do NOT navigate manually — the `onAuthStateChange` stream will fire `signedIn` and go_router redirect handles navigation
- Error handling: catch `AuthException`, show snackbar. Common: "User already registered"
- "Already have an account? Log in" `TextButton` — navigates to `/login`

**3. Create `ForgotPasswordScreen` (`lib/features/auth/presentation/screens/forgot_password_screen.dart`):**
- Email `TextFormField` with validation
- "Send Reset Email" button — calls `ref.read(authRepositoryProvider).resetPasswordForEmail(email)`
- On success: show a `SnackBar` saying "Check your email for a password reset link" and navigate back to `/login`
- On error: show error snackbar

**4. Create `ResetPasswordScreen` (`lib/features/auth/presentation/screens/reset_password_screen.dart`):**
- New password field + confirm password field with validation (min 6 chars, must match)
- "Set New Password" button — calls `ref.read(authRepositoryProvider).updatePassword(newPassword)`
- On success: show snackbar "Password updated successfully", the auth state change will route to home/onboarding
- This screen is shown ONLY when the `passwordRecovery` AuthChangeEvent fires (deep link opens app with recovery token). Route registration happens in Plan 02-02.

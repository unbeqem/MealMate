**`lib/app/router.dart`:**
Follow the exact pattern from RESEARCH.md Pattern 3. Key requirements:
1. `redirect` callback checks `authStateProvider` — if `AsyncLoading` or `AsyncError`, return `null` (no redirect, wait for auth to resolve)
2. If not authenticated and not on `/login`, redirect to `/login`
3. If authenticated and on `/login`, redirect to `/home`
4. `refreshListenable` must watch auth state changes so the router re-evaluates redirect on login/logout

```dart
import 'package:flutter/material.dart';
import 'package:flutter_riverpod/flutter_riverpod.dart';
import 'package:go_router/go_router.dart';
import '../features/auth/presentation/auth_notifier.dart';
import '../features/auth/presentation/screens/login_screen.dart';
import '../features/home/presentation/screens/home_screen.dart';

final routerProvider = Provider<GoRouter>((ref) {
  final authState = ref.watch(authStateProvider);

  return GoRouter(
    initialLocation: '/login',
    redirect: (context, state) {
      // CRITICAL: Don't redirect while auth state is loading (Pitfall 5)
      if (authState.isLoading || authState.hasError) return null;

      final isAuthenticated = authState.valueOrNull?.session != null;
      final isGoingToLogin = state.matchedLocation == '/login';

      if (!isAuthenticated && !isGoingToLogin) return '/login';
      if (isAuthenticated && isGoingToLogin) return '/home';
      return null;
    },
    routes: [
      GoRoute(
        path: '/login',
        builder: (_, __) => const LoginScreen(),
      ),
      GoRoute(
        path: '/home',
        builder: (_, __) => const HomeScreen(),
      ),
    ],
  );
});
```

Note: The `ref.watch(authStateProvider)` inside the Provider automatically rebuilds the GoRouter when auth state changes, serving as the "refreshListenable" equivalent. If this approach causes issues (GoRouter instance recreation), switch to the `refreshListenable` approach from RESEARCH.md using a `ChangeNotifier` adapter.

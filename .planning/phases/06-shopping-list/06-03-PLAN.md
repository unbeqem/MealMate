       - `toggleChecked(String id, bool current)` — calls dao.toggleChecked

       - `updateQuantity(String id, double quantity)` — calls dao.updateQuantity

       - `generateFromMealPlan(String userId)` — calls `ref.read(shoppingListRepositoryProvider).generateFromMealPlan(userId)`. The `shoppingListRepositoryProvider` is defined in `meal_mate/lib/features/shopping/data/shopping_list_repository_provider.dart` (created in plan 06-02).
       All mutations write to Drift; the StreamProvider re-emits automatically. Do NOT manually invalidate or refresh the stream provider.
       Follow 06-RESEARCH.md Pattern 3.

    3. Create `shopping_item_tile.dart` — a `ConsumerWidget` rendering a `CheckboxListTile`:
       - Checkbox toggles `isChecked` via notifier.toggleChecked
       - Title: ingredient name (strikethrough + dimmed if isChecked)
       - Subtitle: quantity + unit formatted via `ref.read(unitNormalizerProvider).formatBaseQuantity(item.quantity, item.family)` (unitNormalizerProvider from plan 06-01)
       - Trailing: delete IconButton calling notifier.removeItem
       - Long-press or tap on quantity: opens inline editing dialog for quantity (SHOP-07)
       Follow 06-RESEARCH.md Pattern 6.

    4. Create `shopping_list_screen.dart`:
       - Scaffold with AppBar title "Shopping List"
       - Body: `ref.watch(shoppingListProvider).when(data:, loading:, error:)`

       - Empty state: when items list is empty, show centered text "No items yet. Regenerate from your meal plan or add items manually."
       - Two FloatingActionButtons (or a Column of FABs):
         a. "Regenerate" button (refresh icon) — calls notifier.generateFromMealPlan(userId)
         b. "Add" button (plus icon) — opens AddItemBottomSheet
       - Show a count badge or subtitle in AppBar: "{N} items ({M} checked)"

    5. Run `dart run build_runner build --delete-conflicting-outputs` to generate Riverpod provider code.

    IMPORTANT: For the quantity edit dialog (SHOP-07), use a simple AlertDialog with a TextFormField for the new quantity value. On submit, call notifier.updateQuantity(id, newQuantity). Keep it simple — no complex stepper widget needed.

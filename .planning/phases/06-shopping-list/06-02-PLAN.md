  <action>
    1. Create `ShoppingListDao` at `meal_mate/lib/features/shopping/data/shopping_list_dao.dart` as a `@DriftAccessor` on `ShoppingListItems` table. Follow the exact pattern from 06-RESEARCH.md Pattern 1:
       - `watchAll(String userId)` -> `Stream<List<ShoppingListItem>>` ordered by isChecked ASC then name ASC
       - `upsertItem(ShoppingListItemsCompanion item)` -> `insertOnConflictUpdate`
       - `deleteItem(String id)` -> delete where id matches
       - `toggleChecked(String id, bool current)` -> update isChecked to !current
       - `updateQuantity(String id, double quantity)` -> update quantity field
       - `replaceAll(List<ShoppingListItemsCompanion> items, String userId)` -> wrapped in `db.transaction()`: DELETE all non-manual items for userId, then INSERT all new items. MUST be transactional per 06-RESEARCH.md Pitfall 4.

    2. Register `ShoppingListDao` in AppDatabase's `@DriftDatabase(tables: [...], daos: [ShoppingListDao])` annotation. Ensure it's accessible as `db.shoppingListDao`.

    3. Create `ShoppingListRepository` at `meal_mate/lib/features/shopping/data/shopping_list_repository.dart`. Constructor takes `AppDatabase` and `UnitNormalizer`. Implement `generateFromMealPlan(String userId)`:
       - Query all `meal_plan_slots` for userId where recipeId is not null
       - For each slot, load the `cached_recipes` row by recipeId
       - Parse recipe JSON, extract `extendedIngredients` (each has `name`, `amount`, `unit`)
       - For each ingredient: call `_normalizer.normalize(quantity: amount, unit: unit)`
       - Build merge key as `"$normalizedName|$family"` — ingredient name lowercased + trimmed + unit family
       - If merge key exists in map: sum baseQty. If not: create new entry.
       - If normalize() returns null (unknown unit): treat as count family, preserve original unit and quantity
       - Convert aggregated map to `ShoppingListItemsCompanion` list with UUID ids, isManual: false
       - Call `dao.replaceAll(companions, userId)` to persist

    4. Run build_runner to generate the DAO mixin.

    CRITICAL: The replaceAll transaction must DELETE only isManual=false items. Manual items added by the user must survive regeneration. This is the core correctness guarantee for SHOP-04 coexisting with SHOP-01.

    CRITICAL: Unknown units from Spoonacular (head, stalk, sprig, bunch, fillet) must NOT be dropped. Preserve them as count family per 06-RESEARCH.md Pitfall 3.

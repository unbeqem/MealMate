<feature>
  <name>Unit Normalization Service (5-Stage Pipeline)</name>
  <files>meal_mate/lib/core/services/unit_normalizer.dart, meal_mate/lib/core/services/unit_normalizer_provider.dart, meal_mate/test/core/services/unit_normalizer_test.dart</files>
  <behavior>
    Stage 1 - Lexical normalization:
    - normalize(quantity: 2.0, unit: "  Cups  ") returns volume family result (case-insensitive, trimmed)
    - normalize(quantity: 1.0, unit: "TBSP") returns volume family result
    - normalize(quantity: 1.0, unit: "Grams") returns mass family result

    Stage 2 - Unit family detection:
    - "g", "kg", "oz", "lb", "lbs", "pound", "pounds", "ounce", "ounces" -> mass family
    - "ml", "l", "tsp", "tbsp", "cup", "cups", "fl oz", "pint", "quart", "gallon" -> volume family
    - "", "whole", "clove", "piece", "slice", "large", "can", "package", "serving" -> count family
    - Unknown strings (e.g., "head", "stalk", "sprig", "bunch", "fillet") -> returns null (caller treats as count)

    Stage 3 - Intra-family conversion to base unit:
    - normalize(quantity: 1.0, unit: "kg") -> baseQty: 1000.0, baseUnit: "g", family: "mass"
    - normalize(quantity: 1.0, unit: "lb") -> baseQty: 453.592, baseUnit: "g", family: "mass"
    - normalize(quantity: 1.0, unit: "cup") -> baseQty: 236.588, baseUnit: "ml", family: "volume"
    - normalize(quantity: 1.0, unit: "tbsp") -> baseQty: 14.7868, baseUnit: "ml", family: "volume"
    - normalize(quantity: 1.0, unit: "tsp") -> baseQty: 4.92892, baseUnit: "ml", family: "volume"
    - normalize(quantity: 2.0, unit: "oz") -> baseQty: 56.699, baseUnit: "g", family: "mass"
    - normalize(quantity: 1.0, unit: "l") -> baseQty: 1000.0, baseUnit: "ml", family: "volume"

    Stage 4 - Cross-family conflict:
    - This stage is handled by the CALLER (repository), not this service. The service returns family info so the caller can build merge keys as "name|family".

    Stage 5 - formatBaseQuantity display:
    - formatBaseQuantity(1500.0, "mass") -> "1.5 kg"
    - formatBaseQuantity(500.0, "mass") -> "500 g"
    - formatBaseQuantity(1500.0, "volume") -> "1.5 L"
    - formatBaseQuantity(236.588, "volume") -> "1 cups" (or similar human-readable)
    - formatBaseQuantity(14.7868, "volume") -> "1 tbsp" (or similar)
    - formatBaseQuantity(3.0, "count") -> "3 whole"

    Edge cases:
    - Empty string unit -> count family, quantity preserved
    - null/unknown unit -> returns null
    - Zero quantity -> works (returns 0 in base unit)
    - Very large quantities -> no overflow (e.g., 10000g -> works)
  </behavior>
  <implementation>
    Create `UnitNormalizer` class as specified in 06-RESEARCH.md Pattern 4:
    - Private `_UnitDef` class with `family` (String) and `factor` (double)
    - Static `_unitMap` mapping ~40 unit aliases to `_UnitDef` entries
    - `_lexNormalize()` for lowercase + trim
    - `normalize({required double quantity, required String unit})` returning nullable record `({String family, double baseQty, String baseUnit})`
    - `formatBaseQuantity(double baseQty, String family)` returning human-readable display string
    - Three unit families: mass (base: g), volume (base: ml), count (no conversion)

    Follow the exact code structure from 06-RESEARCH.md. Do NOT use any external package for unit conversion.

    After TDD cycle completes and all tests pass, create the Riverpod provider file at `meal_mate/lib/core/services/unit_normalizer_provider.dart`:
    ```dart
    import 'package:riverpod_annotation/riverpod_annotation.dart';
    import 'package:meal_mate/core/services/unit_normalizer.dart';

    part 'unit_normalizer_provider.g.dart';

    @riverpod
    UnitNormalizer unitNormalizer(Ref ref) => UnitNormalizer();
    ```
    Then run `dart run build_runner build --delete-conflicting-outputs` to generate the `.g.dart` file. This provider is stateless (UnitNormalizer is a pure service) and exists solely so downstream providers (e.g., shoppingListRepositoryProvider) can inject it via `ref.read(unitNormalizerProvider)` consistently with the project's Riverpod DI pattern.
  </implementation>

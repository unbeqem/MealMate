  <name>Task 2: SpoonacularClient, Edge Function proxy, and RecipeRepository with tests</name>
  <files>

  <behavior>
    - SpoonacularClient.complexSearch sends POST to spoonacular-proxy Edge Function with path and params
    - SpoonacularClient.getRecipeInformation fetches detail for a single recipe ID
    - SpoonacularClient.findByIngredients sends comma-separated ingredient names, ranking=1, ignorePantry=true
    - RecipeRepository.searchRecipes returns RecipeSearchResult from API (cache optional for search)
    - RecipeRepository.getRecipeDetail checks Drift cache first; if fresh, returns cached; if stale or missing, fetches from API and upserts to cache
    - RecipeRepository.findByIngredients returns list of RecipeSummary from API
    - Edge Function reads SPOONACULAR_API_KEY from env, constructs GET URL to api.spoonacular.com, returns JSON
    - Edge Function returns 402 status faithfully when Spoonacular quota is exhausted
  </behavior>
  <action>
    1. Create `supabase/functions/spoonacular-proxy/index.ts` — Deno.serve handler that reads `{ path, params }` from request JSON body. Reads `SPOONACULAR_API_KEY` from `Deno.env.get()`. Constructs URL `https://api.spoonacular.com${path}` with apiKey and all params as query string. Fetches with GET. Returns upstream response JSON and status code. On Spoonacular 402, return 402 to the caller (do not swallow it).

    2. Create `lib/features/recipes/data/spoonacular_client.dart` — Class accepting SupabaseClient. Three methods:
       - `complexSearch({String? query, String? cuisine, int? maxReadyTime, String? includeIngredients, int offset=0, int number=20})` — calls `supabase.functions.invoke('spoonacular-proxy', body: {path: '/recipes/complexSearch', params: {...}})`. Do NOT pass `addRecipeInformation=true` (fetch detail lazily to save API points).
       - `getRecipeInformation(int recipeId)` — path `/recipes/$recipeId/information`, params includeNutrition: false.
       - `findByIngredients(List<String> ingredients)` — path `/recipes/findByIngredients`, params ingredients joined by comma, number: 20, ranking: 1, ignorePantry: true.
       Handle 402 responses by throwing a specific `QuotaExhaustedException`.

    3. Create `lib/features/recipes/data/recipe_repository.dart` — Depends on SpoonacularClient and RecipeCacheDao. Methods:
       - `searchRecipes(...)` → calls client.complexSearch, parses to RecipeSearchResult, caches each result as summary-only to Drift, returns result.
       - `getRecipeDetail(int id)` → checks cache via DAO. If cached and fresh, deserializes jsonData to Recipe and returns. If stale or missing, calls client.getRecipeInformation, parses to Recipe, upserts to cache (jsonData = full JSON, isSummaryOnly = false), returns Recipe.
       - `findByIngredients(List<String> ingredientNames)` → calls client.findByIngredients, parses response to List<RecipeSummary>, caches summaries, returns list.
       Create a Riverpod provider for RecipeRepository using @riverpod annotation.

    4. Create `test/features/recipes/data/recipe_repository_test.dart` — Unit tests:
       - Test getRecipeDetail returns cached recipe when cache is fresh (mock DAO returns fresh entry, verify client NOT called)
       - Test getRecipeDetail fetches from API when cache is stale (mock DAO returns stale entry, verify client called, verify upsert called)
       - Test getRecipeDetail fetches from API when cache is empty (mock DAO returns null)
       - Test searchRecipes caches summaries to Drift
       - Test findByIngredients joins ingredient names with commas
       Use mockito or mocktail for mocking SpoonacularClient and RecipeCacheDao.

    5. Run code generation and tests.

    IMPORTANT: The SpoonacularClient uses `supabase.functions.invoke`, NOT Dio directly. The Edge Function is the HTTP boundary. Do not expose the Spoonacular API key anywhere in the Flutter codebase.
  </action>


Fix three connected bugs that make favoriting from search results a no-op (Gaps 5, 6, 7, 8):

**Fix 1 â€” Repository: upsert before toggle (ingredient_repository.dart):**
Current `toggleFavorite` silently returns when ingredient is not in Drift (`getIngredient` returns null). Search results are bare strings never persisted to Drift.

Change `toggleFavorite` to upsert a minimal Ingredient row if the ingredient doesn't exist:
```dart
Future<void> toggleFavorite(String ingredientId, {String userId = '', String name = ''}) async {
  var ingredient = await _local.getIngredient(ingredientId);
  if (ingredient == null) {
    // Ingredient not in Drift yet (e.g. from search suggestions).
    // Create a minimal row so we can toggle its favorite flag.
    ingredient = Ingredient(
      id: ingredientId,
      name: name.isNotEmpty ? name : ingredientId,
      isFavorite: false,
      cachedAt: DateTime.now(),
    );
    await _local.upsert(ingredient, userId: userId);
  }
  final updated = ingredient.copyWith(
    isFavorite: !ingredient.isFavorite,
  );
  await _local.upsert(updated, userId: userId);
}
```
Add the `name` parameter so callers can provide the display name at call time.

**Fix 2 â€” Favorites provider: append new favorites (ingredient_favorites_provider.dart):**
Current optimistic update only maps over existing list â€” if the ingredient isn't already in the list (first-time favorite), it silently returns unchanged.

Update `toggleFavorite` in the provider to handle the "new favorite" case:
```dart
Future<void> toggleFavorite(String ingredientId, {String name = ''}) async {
  final repo = ref.read(ingredientRepositoryProvider);
  final currentList = state.value ?? [];
  final existingIndex = currentList.indexWhere((i) => i.id == ingredientId);

  if (existingIndex >= 0) {
    // Already in list â€” toggle its isFavorite flag (optimistic)
    final updated = List<Ingredient>.from(currentList);
    updated[existingIndex] = updated[existingIndex].copyWith(
      isFavorite: !updated[existingIndex].isFavorite,
    );
    // Remove if unfavoriting
    if (!updated[existingIndex].isFavorite) {
      updated.removeAt(existingIndex);
    }
    state = AsyncData(updated);
  } else {
    // New favorite â€” append to list (optimistic)
    final newIngredient = Ingredient(
      id: ingredientId,
      name: name.isNotEmpty ? name : ingredientId,
      isFavorite: true,
      cachedAt: DateTime.now(),
    );
    state = AsyncData([...currentList, newIngredient]);
  }

  // Persist to Drift
  await repo.toggleFavorite(ingredientId, name: name);

  // Refresh from source of truth
  if (!ref.mounted) return;
  state = AsyncData(await repo.getFavorites());
}
```

**Fix 3 â€” Search screen: pass isFavorite to tiles (ingredient_search_screen.dart):**

```dart
data: (suggestions) {
  if (suggestions.isEmpty) {
    return _buildEmptyOrBrowseState(context);
  }
  final favoriteNames = favoritesAsync.value
      ?.where((f) => f.isFavorite)
      .map((f) => f.name.toLowerCase())

  return ListView.builder(
    itemCount: suggestions.length,
    itemBuilder: (_, i) {
      final name = suggestions[i];

        name: name,
        isSelected: isSelected,
        isFavorite: isFav,

            .read(ingredientFavoritesProvider.notifier)
            .toggleFavorite(name, name: name),
        onSelectTap: () => ref
            .read(selectedTodayProvider.notifier)
            .toggle(name, name: name),
      );
    },
  );
},
```

Note: The `name` parameter must be added to the provider's `toggleFavorite` signature and passed through to the repository's `toggleFavorite`.

**1. Create `OpenFoodFactsRemoteSource` (`lib/features/ingredients/data/openfoodfacts_remote_source.dart`):**

Methods:
- `Future<List<String>> getSuggestions(String query, {int limit = 20})` — calls `OpenFoodAPIClient.getSuggestions(TagType.INGREDIENTS, input: query, language: OpenFoodFactsLanguage.ENGLISH, limit: limit)`. Returns the raw `List<String>`.
- `Future<List<Ingredient>> searchByCategory(String categoryTag)` — calls `OpenFoodAPIClient.searchProducts(null, ProductSearchQueryConfiguration(...))` with `TagFilter.fromType(tagFilterType: TagFilterType.CATEGORIES, tagName: categoryTag)`. Maps each `Product` to an `Ingredient` domain model, parsing `ingredientsAnalysisTags` for vegan/vegetarian and `labelsTags` for gluten-free/dairy-free into the `dietaryFlags` list. Limit to 50 results per call.

Static category map (define as a top-level constant):
```dart
const ingredientCategories = {
  'Produce': 'en:fruits-and-vegetables',
  'Dairy': 'en:dairies',
  'Meat': 'en:meats',
  'Seafood': 'en:seafood',
  'Grains': 'en:cereals-and-potatoes',
  'Legumes': 'en:legumes-and-their-products',
  'Spices': 'en:spices',
  'Condiments': 'en:sauces',
  'Oils': 'en:fats',
  'Beverages': 'en:beverages',
};
```

Dietary flag parsing logic for each Product:
- Check `product.ingredientsAnalysisTags?.veganStatus` — if `VeganStatus.IS_VEGAN` → add "vegan"
- Check `product.ingredientsAnalysisTags?.vegetarianStatus` — if `VegetarianStatus.IS_VEGETARIAN` → add "vegetarian"
- Check `product.labelsTags` contains "en:gluten-free" → add "gluten-free"
- Check `product.labelsTags` contains "en:no-lactose" or "en:dairy-free" → add "dairy-free"

**2. Create `IngredientLocalSource` (`lib/features/ingredients/data/ingredient_local_source.dart`):**

Constructor takes `AppDatabase`. Methods using Drift query API:
- `Future<List<Ingredient>> getIngredientsByCategory(String category)` — SELECT WHERE category = :category
- `Future<Ingredient?> getIngredient(String id)` — SELECT WHERE id = :id
- `Future<void> upsert(Ingredient ingredient)` — INSERT OR REPLACE into ingredients table, encoding `dietaryFlags` as JSON string
- `Future<void> upsertAll(List<Ingredient> ingredients)` — batch upsert
- `Future<List<Ingredient>> filterByDietary(Set<DietaryRestriction> restrictions)` — SELECT WHERE dietaryFlags column contains all requested flags. For each restriction, generate a `dietaryFlags LIKE '%vegan%'` clause. Combine with AND.
- `Future<List<Ingredient>> getFavorites()` — SELECT WHERE isFavorite = true
- `Future<void> addSelectedToday(String ingredientId, String userId)` — INSERT into selected_today_ingredients with today's date
- `Future<void> removeSelectedToday(String ingredientId)` — DELETE from selected_today_ingredients WHERE ingredientId AND date = today
- `Future<List<String>> getSelectedToday(String userId)` — SELECT ingredientId WHERE selectedDate = today AND userId
- `Future<void> clearSelectedToday(String userId)` — DELETE WHERE selectedDate = today AND userId

Convert between domain `Ingredient` and Drift `IngredientsCompanion` in each method. The `dietaryFlags` column stores JSON (e.g., `'["vegan","gluten-free"]'`) — use `jsonEncode`/`jsonDecode`.

**3. Create `IngredientRepository` (`lib/features/ingredients/data/ingredient_repository.dart`):**

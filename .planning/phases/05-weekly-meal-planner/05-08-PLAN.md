Fix in MealPlanRepository.assignRecipe (or the caller): after assigning the recipe to a slot, check if the CachedRecipes entry for that spoonacularId has isSummaryOnly=true. If so, call RecipeRepository.getRecipeInformation(spoonacularId) to fetch the full detail and update the cache with extendedIngredients populated. This is the same call used when viewing recipe detail.

Implementation:
1. MealPlanRepository needs a reference to RecipeRepository (inject via constructor or Ref)
2. In assignRecipe, after the DB insert, check if cached entry is summary-only
3. If summary-only, call getRecipeInformation to backfill the full detail (fire-and-forget or awaited)
4. weekIngredientNamesProvider will then find extendedIngredients and populate the panel

Important: Do NOT remove the isSummaryOnly skip in weekIngredientNamesProvider — that is correct defensive behavior. The fix is to ensure full data is fetched.

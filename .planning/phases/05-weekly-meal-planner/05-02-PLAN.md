  <files>meal_mate/lib/features/meal_planner/presentation/planner_screen.dart, meal_mate/lib/features/meal_planner/presentation/widgets/planner_grid.dart, meal_mate/lib/features/meal_planner/presentation/widgets/meal_slot_card.dart, meal_mate/lib/features/meal_planner/presentation/widgets/empty_slot_card.dart</files>
  <action>
**1. Create `planner_screen.dart`:**
A `ConsumerStatefulWidget` that manages the current `weekStart` state. Layout:
- AppBar with title "Meal Planner"
- Week navigation row: left arrow IconButton, week label (e.g., "Mar 2 - Mar 8, 2026" using `DateFormat`), right arrow IconButton
- `weekStart` state initialized to `weekStartFor(DateTime.now())`
- Left arrow: `setState(() => weekStart = prevWeek(weekStart))`
- Right arrow: `setState(() => weekStart = nextWeek(weekStart))`
- Body: `PlannerGrid(weekStart: weekStart)`
- FloatingActionButton or AppBar action for "Save as Template" (placeholder, wired in 05-04)

Use `weekStartFor()`, `nextWeek()`, `prevWeek()` from the date_utils created in 05-01.

**2. Create `planner_grid.dart`:**
A `ConsumerWidget` that watches `mealPlanNotifierProvider(weekStart)`. Layout:
- `SingleChildScrollView(scrollDirection: Axis.horizontal)` wrapping a `Row` of 7 day columns
- Each day column: a `Column` with a day header (Mon, Tue, ... using `DateFormat('EEE').format(weekStart.add(Duration(days: index)))`) and 3 meal cells (breakfast, lunch, dinner)
- Each cell: either `MealSlotCard` (if slot has recipeId) or `EmptySlotCard` (if empty/null)
- Pass the `MealSlot?` data and callbacks to each cell
- Day column width: fixed ~120px with padding. Meal row labels ("Breakfast", "Lunch", "Dinner") on the left as a fixed column before the scrollable row.

Static constants: `_days = ['monday', 'tuesday', 'wednesday', 'thursday', 'friday', 'saturday', 'sunday']` and `_meals = ['breakfast', 'lunch', 'dinner']`

Handle the `AsyncValue` from the notifier: show `CircularProgressIndicator` on loading, error message on error, grid on data.

**3. Create `meal_slot_card.dart`:**
A stateless widget displaying a filled slot. Shows:
- Recipe thumbnail (if available, use `Image.network` with a placeholder)
- Recipe title (truncated to 2 lines)
- On tap: show a bottom sheet or popup menu with options: "Replace" (opens recipe picker) and "Remove" (calls notifier.clearSlot)
- Sized to fit within the ~120px column width

**4. Create `empty_slot_card.dart`:**
A stateless widget for an empty slot. Shows:
- Dashed border or light grey background with a "+" icon
- Meal type label below the icon (e.g., "Breakfast")
- On tap: opens `RecipePickerSheet` to assign a recipe

Both cards should have a consistent height (~100px) for grid alignment.

**1. Create `recipe_picker_sheet.dart`:**
A bottom sheet widget (`showModalBottomSheet`) that lists cached recipes from Drift for the user to pick. This is the recipe assignment flow for PLAN-02 and the "Replace" action for PLAN-03.

Layout:
- Search bar at top (filters the list by recipe title — local filter on cached data)
- `ListView.builder` of cached recipe cards, each showing title + thumbnail + cook time
- On tap: calls the `onRecipeSelected(int recipeId)` callback passed from the caller
- The caller (EmptySlotCard tap or MealSlotCard "Replace") opens the sheet and passes a callback that calls `notifier.assignRecipe(dayOfWeek: ..., mealType: ..., recipeId: selectedId)`
- Close the sheet after selection via `Navigator.pop(context)`

For the recipe list, create a simple `cachedRecipesProvider` that reads all cached recipes from the Drift `cachedRecipes` table (from Phase 4). If Phase 4's recipe cache table uses a different name or structure, adapt accordingly. The provider should be a simple FutureProvider or StreamProvider watching the cached recipes table.

**2. Wire route in `router.dart`:**
Add a `/planner` route to the go_router configuration that navigates to `PlannerScreen`. This should be a top-level route (or nested under the authenticated shell route if one exists from Phase 2).

If there is a bottom navigation bar scaffold from Phase 2/4, add the planner as a tab. Otherwise, register as a standalone route accessible from the home/dashboard screen.

**IMPORTANT for PLAN-03 (edit/replace/remove):** The MealSlotCard's tap handler must present options:
- "Replace" → opens RecipePickerSheet with the same callback as empty slot
- "Remove" → calls `notifier.clearSlot(dayOfWeek: slot.dayOfWeek, mealType: slot.mealType)`

This can be a `showModalBottomSheet` with two `ListTile` options, or a `PopupMenuButton`.

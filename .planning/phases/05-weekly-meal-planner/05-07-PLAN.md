```dart
class RecipeCard extends StatelessWidget {

  });

```

In the `build` method, change the InkWell's onTap from hardcoded navigation to:
```dart
onTap: onTap ?? () => context.push('/recipes/${recipe.id}'),
```

This preserves existing behavior (detail navigation) when onTap is null, but allows callers to override.

**Step 2 — _SelectableRecipeCard (recipe_browse_screen.dart):**

Remove the `GestureDetector` wrapper (lines 389-396). Instead, pass the pop callback directly to RecipeCard via its new `onTap` parameter:

Replace:
```dart
return GestureDetector(
  onTap: () {
    context.pop<Map<String, dynamic>>({
      'recipeId': recipe.id,
      'recipeTitle': recipe.title,
      'recipeImage': recipe.image,
    });
  },
  child: Stack(

      RecipeCard(recipe: recipe),
```

With:
```dart
return Stack(
  children: [
    RecipeCard(
      recipe: recipe,
      onTap: () {
        context.pop<Map<String, dynamic>>({
          'recipeId': recipe.id,
          'recipeTitle': recipe.title,
          'recipeImage': recipe.image,
        });
      },
    ),
```

This eliminates the double-tap conflict because only one onTap handler exists (the pop), and InkWell uses it instead of its default push.

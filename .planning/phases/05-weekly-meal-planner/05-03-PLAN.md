**1. Create `meal_slot_data.dart`:**
A Freezed data class used as the drag payload. Must be lightweight (no database references):
```dart
@freezed
sealed class MealSlotData with _$MealSlotData {
  const factory MealSlotData({
    required String dayOfWeek,
    required String mealType,
    int? recipeId,
    String? recipeTitle,
    String? recipeImage,
  }) = _MealSlotData;
}
```

**2. Wrap `MealSlotCard` in `LongPressDraggable<MealSlotData>`:**
- `data`: MealSlotData from the slot's current state
- `feedback`: A `Material(elevation: 4)` widget showing a compact recipe chip (title + small image), sized ~120x80px. This is what the user sees while dragging.
- `childWhenDragging`: An `_EmptySlotPlaceholder` or semi-transparent version of the card (indicates the source slot is temporarily "empty")
- `child`: The existing MealSlotCard widget
- Use `LongPressDraggable` (NOT `Draggable`) to disambiguate from tap gestures (tap = edit/replace, long-press = drag)

**3. Wrap ALL cells (filled AND empty) in `DragTarget<MealSlotData>`:**
In `planner_grid.dart`, each `_MealSlotCell` widget should be wrapped in a `DragTarget<MealSlotData>`:
- `onWillAcceptWithDetails`: Return true unless the dragged item is being dropped on its own source slot (same dayOfWeek AND mealType)
- `onAcceptWithDetails`: Call `ref.read(mealPlanNotifierProvider(weekStart).notifier).swapSlots(from, to)` where `from` is `details.data` converted to a MealSlot, and `to` is the target cell's MealSlot. For drops on empty slots, this effectively moves the recipe (source gets null, target gets the recipe).
- `builder`: Use `candidateItems.isNotEmpty` to show a highlight border (e.g., `Border.all(color: colorScheme.primary, width: 2)`) on the target cell when a drag is hovering. Use `AnimatedContainer` with a short duration (150ms) for smooth transitions.

**4. Handle scroll-drag conflict:**
The grid uses `SingleChildScrollView(scrollDirection: Axis.horizontal)`. This conflicts with `LongPressDraggable` (see research Pitfall 1). Mitigate with:
- Add a `ValueNotifier<bool> _isDragging` to the PlannerGrid or its parent
- On `LongPressDraggable.onDragStarted`: set `_isDragging.value = true`
- On `LongPressDraggable.onDragEnd` and `onDraggableCanceled`: set `_isDragging.value = false`
- Wrap `SingleChildScrollView` in a `ValueListenableBuilder<bool>` that switches physics:
  - `_isDragging == true` → `NeverScrollableScrollPhysics()`
  - `_isDragging == false` → `AlwaysScrollableScrollPhysics()` or default

This disables horizontal scrolling during a drag operation and re-enables it when the drag completes or is canceled.

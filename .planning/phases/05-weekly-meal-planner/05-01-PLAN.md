**1. Create `meal_plan_dao.dart`:**
A Drift DAO that encapsulates the join query between `mealPlanSlots` and `cachedRecipes` (from Phase 4). The DAO provides:
- `watchSlotsForWeek(String userId, DateTime weekStart)` — returns `Stream<List<MealSlot>>` via a `select().join([leftOuterJoin(cachedRecipes, ...)]).watch()`
- `upsertSlot(MealPlanSlotsCompanion companion)` — `insertOnConflictUpdate` targeting the composite key
- `clearSlot(String userId, DateTime weekStart, String dayOfWeek, String mealType)` — update setting recipeId to null
- `swapSlots(MealSlot from, MealSlot to)` — wrapped in `db.transaction()`

The join maps `cachedRecipes.title` and `cachedRecipes.thumbnailUrl` (or equivalent column name from Phase 4) to the `MealSlot.recipeTitle` and `MealSlot.recipeImage` fields.

**2. Create `meal_plan_repository.dart`:**

- `assignRecipe({dayOfWeek, mealType, recipeId, weekStart})` — builds companion, calls DAO.upsertSlot

**3. Create `meal_plan_notifier.dart`:**
```dart
@riverpod
class MealPlanNotifier extends _$MealPlanNotifier {
  @override
  Stream<List<MealSlot>> build(DateTime weekStart) {
    final repo = ref.watch(mealPlanRepositoryProvider);
    return repo.watchWeek(weekStart);
  }

  Future<void> assignRecipe({required String dayOfWeek, required String mealType, required int recipeId}) async {
    final repo = ref.read(mealPlanRepositoryProvider);
    await repo.assignRecipe(dayOfWeek: dayOfWeek, mealType: mealType, recipeId: recipeId, weekStart: state.value!.first.weekStart);
  }

  Future<void> clearSlot({required String dayOfWeek, required String mealType}) async {
    final repo = ref.read(mealPlanRepositoryProvider);
    await repo.clearSlot(dayOfWeek: dayOfWeek, mealType: mealType, weekStart: state.value!.first.weekStart);
  }

  Future<void> swapSlots(MealSlot from, MealSlot to) async {
    final repo = ref.read(mealPlanRepositoryProvider);
    await repo.swapSlots(from, to);
  }
}
```

Create a `mealPlanRepositoryProvider` Riverpod provider that instantiates MealPlanRepository with the AppDatabase and current user ID.

**4. Run codegen:** `dart run build_runner build --delete-conflicting-outputs`

**IMPORTANT:** The swapSlots method MUST use `db.transaction()` to wrap both update statements. Without a transaction, a crash between the two writes leaves the database in an inconsistent state (research Pitfall 4).

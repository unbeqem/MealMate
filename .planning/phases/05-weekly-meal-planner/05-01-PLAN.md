CRITICAL TYPE MISMATCH: MealPlanSlots.recipeId is TextColumn referencing Recipes.id (UUID text PK), but actual recipe data lives in CachedRecipes which uses IntColumn id (Spoonacular integer PK). The meal planner stores Spoonacular recipe IDs. Solution: store the Spoonacular integer ID as its string representation in the existing recipeId text column (e.g., "716429"). Join with CachedRecipes by casting: `db.cachedRecipes.id.equals(int.parse(slot.recipeId))` or use `customExpression` for the join. This avoids a column type migration while keeping the existing FK declaration (SQLite does not enforce FK type matching at runtime when PRAGMA foreign_keys checks existence, not type).
